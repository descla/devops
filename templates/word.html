一、新建系统

1. 适用场景

该变更适用于

（1）x86服务器系统安装变更

2. 变更步骤

2.1. Windows

①　登录x86服务器带外管理web页面，选择虚拟控制台

②　挂载windows自动安装镜像

③　启动服务器，安装系统

2.2. Linux

①　登录x86服务器带外管理web页面，选择虚拟控制台

②　挂载红帽Linux ks自动安装镜像

③　启动服务器，安装系统

3. 回退方案

以上系统变更不存在回退问题。

4. 应急预案

安装过程中若出现硬件告警，终止安装，并联系服务器厂商进行维修。

5. 服务负责人管理意见

系统安装是项目上线和系统节点扩容的基本变更，系统的安装是依据x86服务器安装标准，定制的自动化安装镜像，无风险，建议纳入标准变更。

6. 部门经理审批意见

 

二、用户

1.  适用场景

该变更适用于

（1）日常的x86系统用户常规变更，包括用户增加、删除、用户口令、用户组等参数修改。

2.  变更步骤

2.1. Linux

增加用户

useradd  <username>

删除用户

userdel  -r  <username>

修改用户口令

passwd

增加用户组

groupadd  <groupname>

删除用户组

groupdel  <groupname>

将用户加入到用户组中

usermod  -aG  <groupname>  <username>

将用户从用户组中删除

gpasswd  <groupname>  -d  <username>

2.2. Windows

增加用户

net  user  <username>  <password>（可选） /add

删除用户

net  user  <username>  /del

修改用户口令

net  user  <username>  <newpassword>

增加用户到指定的组

net  localgroup  <groupname>  <username>  /add 

 

3.  回退方案

以上系统变更不存在回退问题。

4.  应急预案

提前备份/etc/passwd和/etc/shadow文件，如果修改用户密码导致问题，使用/etc/passwd和/etc/shadow文件恢复配置。

5.  服务负责人管理意见

用户常规变更一般由系统功能调整、用户状态、权限变化等需求引起，是支持系统稳定运行常态化、低风险的操作，建议纳入标准变更。

6.  部门经理审批意见

 

三、目录及权限

1．适用场景

该变更适用于

（1）日常的x86 Linux系统目录（文件夹）变更，包裹新建目录，移动目录，更改目录名等

（2）日常的x86 Linux系统权限变更，包括修改目录权限、修改目录所属组、修改文件权限及所属组等参数修改。

2．变更步骤

2.1. Linux

新建目录

mkdir  </path/to/dirname>

修改目录权限

chmod  <modnumber> </path/to/fileordir>

修改目录所属组

chown  <user>:<group> </path/to/fileordir>

3．回退方案

以上系统变更不存在回退问题。

4．应急预案

提前记录目录修改前权限，如果修改权限后导致问题，使用chmod命令恢复目录权限。

5．服务负责人管理意见

目录及权限变更一般由目录结构调整、文件权限变化等需求引起，是支持系统稳定运行常态化、低风险的操作，建议纳入标准变更。

6．部门经理审批意见

四、文件系统及swap

1．适用场景

该变更适用于

（1）日常的x86 Linux系统文件系统扩容的变更，包括逻辑卷扩容、数据库RAC表空间扩容、挂载文件系统扩容、挂载文件系统等

（2）日常的x86 Linux系统swap扩容的常规变更，包括SWAP扩容、SWAP空间清理等。

2．变更步骤

2.1. 文件系统扩容

发现新增存储

echo “- - -” > /sys/class/scsi_host/host*/scan

给新增存储制作PV

pvcreate  /dev/<diskname>

扩容逻辑卷组

vgextend <vgname> /dev/<diskname>

扩容逻辑卷

Lvextend -L +10G /dev/mapper/<vgname>-<lvname>

扩容文件系统

resize2fs  /dev/mapper/<vgname>-<lvname>

2.2. swap扩容

发现新增存储

echo “- - -” > /sys/class/scsi_host/host*/scan

给新增存储制作PV

pvcreate  /dev/<diskname>

扩容逻辑卷组

vgextend <vgname> /dev/<diskname>

扩容逻辑卷

lvextend -L +10G /dev/mapper/rootvg-swaplv

取消swap挂载

swapoff -a

制作swap卷

mkswap /dev/mapper/rootvg-swaplv

重新挂载swap

swapon -a

 

3．回退方案

逻辑卷扩容通过lvm的归档功能来记录操作，若扩容失败可将逻辑卷状态回退到扩容的状态。

4．应急预案

如扩容失败，并影响了以前的操作系统，将扩容的文件系统挂载为只读状态，并将数据拷出故障盘。

5．服务负责人管理意见

文件系统和swap扩容一般由磁盘使用空间不足、数据库表空间扩容、swap空间利用率过高等需求引起，是支持系统稳定运行常态化、低风险的操作，建议纳入标准变更。

6．部门经理审批意见

五、共享目录挂载

 

1．适用场景

该变更适用于

(1)     日常的x86 Linux nfs共享目录远程挂载

(2)     日常的x86 Linux 共享NAS远程挂载。

2．变更步骤

发现共享目录

showmount -e <ipaddr>

挂载共享目录

mount -t nfs <ipaddr>:</path/to/sharedir> </mount/dir>

写入fstab文件保证开机自动挂载

echo “<ipaddr>:</path/to/sharedir> </mount/dir> nfs defaults 0 0” >> /etc/fstab

 

3．回退方案

取消共享目录的挂载,umount <mountdir>

4．应急预案

如挂载失败，联系存储管理员确认共享目录路径和ip是否正确，是否开放acl访问权限。必要时可联系NAS厂商技术支持

5．服务负责人管理意见

挂载共享目录或共享NAS一般是由应用方提出的为实现应用集群化的资源需求，可提高系统运行的高可用性、是低风险的操作，建议纳入标准变更。
